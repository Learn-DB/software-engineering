# @Transactional(readOnly = true)

## 의미
- Hibernate/JPA: Flush 모드를 `MANUAL` 로 변경해 쓰기 시도를 최소화하고, 일부 DB 드라이버는 읽기 전용 힌트를 전달한다.
- DB: MySQL/InnoDB 등 일부 데이터베이스는 세션을 읽기 전용으로 표시해 DML 수행 시 예외를 던질 수 있다.

## 효과
- 변경 감지(Dirty Checking)가 비활성화되어 트랜잭션 종료 시 flush 비용이 줄어든다.
- 지연 로딩은 가능하지만, 엔티티 변경사항이 자동으로 반영되지 않으므로 DTO 변환 전 읽기 전용 패턴에 적합하다.

## 주의 사항
- 메서드 내에서 엔티티를 수정해도 커밋되지 않으므로 실수로 데이터 손실을 막을 수 있지만, 변경이 필요한 경우 별도 트랜잭션을 사용해야 한다.
- 일부 JDBC 드라이버는 실제로 DML을 막지 않으므로 테스트로 검증 필요.
- `readOnly=true` 인 트랜잭션에서 다른 쓰기 트랜잭션 메서드를 호출해도 상위 트랜잭션 설정이 유지된다 (전파 수준 확인 필요).

## 활용 예시
- 조회 전용 API, 검색, 보고서 생성.
- 배치 작업에서 대량 읽기 단계와 쓰기 단계를 분리해 성능을 최적화.
