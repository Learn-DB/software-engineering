# 인덱스 기본 개념

## 목적
- 특정 컬럼 값을 빠르게 찾기 위해 정렬/구조화된 데이터 구조(B-Tree, Hash 등)를 유지한다.
- 전체 테이블 스캔을 줄여 지연 시간, I/O 비용을 낮춘다.

## 구조 유형
- **B-Tree 인덱스**: 대부분의 RDB 기본 구조. 범위 검색, 정렬, LIKE 접두사 검색에 적합.
- **Hash 인덱스**: 동등 비교에 특화. 범위 조건에는 부적합.
- **GiST/GIN**: PostgreSQL의 특수 인덱스. 공간 검색, 배열/JSON 쿼리에 사용.
- **Full-text 인덱스**: 텍스트 검색을 위한 역색인 구조.

## 설계 원칙
- 선택도(Selectivity)가 높은 컬럼부터 구성한다.
- 다중 컬럼 인덱스는 조건절과 정렬 순서를 모두 고려해 순서를 정한다.
- 쓰기 집중 테이블에서는 인덱스 수를 최소화하고, 배치/쓰기 시간과의 트레이드오프를 기록한다.

## 관리
- 통계 갱신(`ANALYZE`)을 자동화해 옵티마이저가 최신 정보를 사용하도록 한다.
- 사용되지 않는 인덱스를 정기적으로 분석하고 제거해 저장공간/쓰기 비용을 줄인다.
- 파티셔닝/샤딩 환경에서는 로컬/글로벌 인덱스 전략을 문서화한다.
