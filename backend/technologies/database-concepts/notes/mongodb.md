# MongoDB 개요

## 특징
- 문서 지향 NoSQL DB. BSON 포맷으로 유연한 스키마를 지원.
- 샤딩과 복제 세트를 통해 수평 확장과 고가용성을 제공.
- 4.0 이후 다큐먼트 단위 ACID 트랜잭션을 지원하며, 4.2부터는 샤드 간 트랜잭션도 가능.

## 장점
- 스키마 변경 비용이 낮아 빠른 프로토타이핑에 적합.
- 애그리게이션 파이프라인, Atlas Search 등 풍부한 쿼리/검색 기능.
- Geospatial/TTL/Partial 인덱스 등 특수 인덱스를 손쉽게 추가.

## 주의 사항
- 다큐먼트 크기 제한(16MB)을 넘어가면 GridFS 사용 필요.
- 조인이 제한적이므로, 데이터 모델링 시 중첩 vs 참조 전략을 명확히 선택한다.
- 쓰기 Concern/Read Preference 설정으로 일관성 수준을 조정해야 한다.

## 운영 팁
- 샤드 키를 신중히 선택하여 편향을 방지하고, Balancer 상태를 모니터링한다.
- WiredTiger 캐시, 커넥션 풀, Oplog 크기 등을 모니터링하고 알람을 설정한다.
- Schema Versioning을 애플리케이션에서 관리하여 호환성을 유지한다.
