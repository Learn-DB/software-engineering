# 파티셔닝 (Partitioning)

단일 DB 인스턴스 내에서 테이블/인덱스를 물리적으로 여러 파티션으로 나눠 I/O와 관리성을 높인다.

## 유형
- **Range Partition**: 날짜, 숫자 범위로 구간을 나눈다. 대부분의 아카이빙 용례에 적합.
- **List Partition**: 특정 값 집합(지역 코드 등)에 따라 분리.
- **Hash Partition**: 해시 함수로 균등 분배해 단일 파티션 수평 확장.
- **Composite**: Range + Hash 등 복합 분할로 편향과 병렬성을 동시에 잡는다.

## 장점
- 오래된 파티션을 `DETACH/DROP` 하여 빠르게 아카이빙 가능.
- 파티션 프루닝을 통해 필요한 파티션만 스캔 → 쿼리 성능 향상.
- 유지보수/리빌드 작업을 파티션 단위로 수행해 다운타임 최소화.

## 주의 사항
- 파티션 키가 WHERE 절에 포함되지 않으면 프루닝 이점을 못 얻는다.
- 파티션 수가 너무 많으면 계획 수립/메타데이터 관리 비용 증가.
- 글로벌 인덱스(모든 파티션 공유)가 필요한 DB인지 확인.

## 운영 체크리스트
- 파티션 규칙과 보존 기간(Retention)을 문서화해 자동화 스크립트로 관리.
- 신규 파티션 생성 스케줄을 잡고 모니터링 알람을 설정한다.
- 파티션간 데이터 이동 시 트랜잭션 로그와 복제를 고려한다.
