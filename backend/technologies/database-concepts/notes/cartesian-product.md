# 카테시안 곱 (Cartesian Product)

두 테이블을 조건 없이 조인할 때 발생하는 모든 행의 조합으로, 행 수가 급격히 증가한다.

## 특징
- `SELECT * FROM A, B` 혹은 `CROSS JOIN` 이 기본적으로 카테시안 곱이다.
- 실수로 조인 조건을 빠뜨린 `INNER JOIN` 도 동일한 효과를 내며, 성능/결과 오류를 야기한다.

## 영향
- 결과 셋이 `|A| x |B|` 규모가 되므로 메모리, 네트워크, 애플리케이션 부담이 급격히 늘어난다.
- WHERE 절이 뒤늦게 필터링하더라도 조인 수행 비용이 이미 발생한다.

## 예방
- ERD 기반으로 조인 키를 항상 명시하고, ORM 관계 매핑도 필수 조인 조건을 강제하도록 설정한다.
- 코드 리뷰/SQL 린트에서 `CROSS JOIN` 혹은 조인 조건 누락을 검사한다.
- 분석 쿼리에서 의도적으로 사용하는 경우 주석과 제한 조건을 문서화해 재사용 시 오해를 줄인다.
