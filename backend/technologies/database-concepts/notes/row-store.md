# 행 기반 DB (Row-oriented Store)

## 특징
- 레코드 단위로 디스크에 연속 저장. 하나의 행에 속한 모든 컬럼을 같이 읽고 쓴다.
- 트랜잭션 처리(OLTP)와 같이 한 행 전체를 자주 수정하는 워크로드에 최적화.

## 장점
- INSERT/UPDATE/DELETE 시 관련 컬럼을 한 번에 접근하므로 쓰기 성능이 우수.
- PK 기반 조회, 소량 컬럼만 갖는 좁은 인덱스 사용 시 간단.
- RDB 대부분(MySQL, PostgreSQL, SQL Server 등)이 행 저장 구조를 사용.

## 단점
- 대규모 집계/분석에서 불필요한 컬럼까지 읽게 되어 I/O 낭비.
- 압축률이 낮고, 캐시 효율이 떨어질 수 있음.

## 활용
- 주문/결제 등 강한 일관성과 다수의 개별 트랜잭션을 처리하는 시스템.
- CQRS 패턴에서 Write 모델로 사용하고, 읽기 전용 모델은 컬럼 저장소나 검색엔진으로 보조.
