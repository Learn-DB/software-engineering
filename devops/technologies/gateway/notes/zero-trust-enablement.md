# 인에이블먼트 제로 트러스트 (Zero Trust Enablement)

## 개념
- 네트워크 경계가 아닌 사용자와 디바이스의 신뢰를 지속적으로 검증하여 접근을 허용하는 보안 모델.
- "절대 신뢰하지 말고 항상 검증하라" 원칙에 따라 모든 요청에 대해 신원, 기기 상태, 컨텍스트를 평가한다.

## 구현 구성요소
1. **ID 및 액세스 관리**: SSO, MFA, Just-In-Time 권한 부여.
2. **디바이스 보안**: EDR/MDM 상태 검사, 컴플라이언스 태그.
3. **네트워크 제어**: mTLS, 소프트웨어 정의 경계(SDP), 정책 기반 마이크로 세그멘테이션.
4. **정책 엔진**: 사용자/디바이스/리소스 컨텍스트를 기반으로 허용/차단/추가 검증 결정.
5. **관측성**: 모든 요청을 로깅하여 이상 징후 탐지, 실시간 대응 자동화.

## Enablement 전략
- 개발/운영팀에 Zero Trust 정책 템플릿을 제공하고, 서비스 온보딩 시 체크리스트를 완료하도록 한다.
- 게이트웨이/프록시 계층에 정책 엔진을 통합해 인증/인가/세션 지속 검사를 자동화.
- 교육 프로그램을 운영하여 최소 권한 원칙, 기기 보안, 정책 예외 신청 절차를 전사적으로 이해시킨다.
- 파일럿 → 단계별 확장 계획을 세워 레거시 네트워크 의존성을 점진적으로 제거.

## 모니터링/개선
- 정책 허용/차단 비율, 사용자 불편 지표, 보안 사건 대응 시간을 추적.
- 정책 변경은 GitOps 파이프라인으로 버전 관리하여 감사 가능성을 확보.
- 주기적으로 Red Team/Blue Team 연습을 통해 Zero Trust 제어가 제대로 작동하는지 검증.
